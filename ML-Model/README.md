# ATLAS ML Pipeline: AIOps & Anomaly Detection

## Project Overview
This module represents the core Machine Learning intelligence layer for the ATLAS platform. It is designed to run anomaly detection (using models like Isolation Forest) to mathematically detect sophisticated, multi-metric causal anomalies (thermal cascades, crypto-miners, memory leaks, etc.) across tens of thousands of servers.

The mathematical predictions (health scores, anomaly markers) are appended to a lean hardware context schema and stored directly in ClickHouse. A downstream Small Language Model (SLM) then reads this ClickHouse context window to generate automated, natural language Root Cause Analysis (RCA) reports for system operators.

---

## Data Schemas

### 1. Raw Input Schema (31 Fields)
Generated by `data_generator.py` and `live_data_gen.py`. This contains the full hardware context, customer IDs, and deep technical metrics required for the ML model to perform accurate feature engineering.

* **Identifiers:** `report_id`, `device_id`, `server_name`, `application_customer_id`, `platform_customer_id`, `tags`
* **Geography:** `location_name`, `location_city`, `location_state`, `location_country`
* **Hardware:** `processor_vendor`, `server_generation`, `cpu_inventory`, `memory_inventory`, `pcie_devices_count`, `socket_count`
* **Temporal:** `last_maintenance_date`, `last_boot_time`, `metric_time`
* **Physics/Telemetry:** `avg_metric_value`, `max_metric_value`, `min_metric_value`, `cpu_utilization`, `memory_utilization`, `disk_utilization`, `network_throughput`, `cpu_temperature`, `amb_temp`, `fan_speed_rpm`, `gpu_utilization`
* **Labels:** `is_anomaly` (stripped during training)

### 2. SLM Context Schema (ClickHouse Output)
The highly-optimized schema written to ClickHouse *after* ML inference. Heavy categorical UUIDs (like customer IDs) are deliberately dropped to save SLM context tokens, while the critical hardware context and new AI predictions are preserved for Root Cause Analysis (RCA).

## Quickstart

### 1. Boot the ML Container

Run the following command from the root `atlas/` directory:

```bash
docker compose up -d atlas-ml --build
```

This starts the isolated ML container and automatically initializes the live inference pipeline.

---

### 2. Generate Historical Training Data (One-Time Setup)

Generate the historical telemetry dataset used for model training.

```bash
docker exec -it atlas-ml python data_generator.py
```

This creates approximately 30 days of synthetic telemetry data (configurable through the `--days` argument), automatically partitioned into:

### 3. Train the Isolation Forest Model (one-time)

Train the anomaly detection model using the generated historical dataset.

```bash
docker exec -it atlas-ml python train_model.py
```

The training pipeline automatically:

* Loads all training Parquet files from training set 
* Performs feature engineering
* Builds the preprocessing pipeline
* Trains the Isolation Forest model
* Computes anomaly score thresholds
